GENAI MENTOR - CURRENT BUILD SUMMARY
====================================

Status
------
The project is a working educational GenAI system with:
- A deployed Streamlit app at https://studybuddygenai.streamlit.app/
- A local Streamlit entry point in app.py
- Student chat and Backend Tracking views
- Hybrid offline/online RAG
- LangGraph-compatible multi-agent orchestration
- Prompt templates, safety checks, tools, evaluation outputs, and trace logging
- Qwen/Qwen2.5-0.5B-Instruct LoRA fine-tuning artifacts
- Runtime model selection for Salma `_salma` adapters, Fatma `_fatma` adapters, Khadija `_khadija` adapters, base Qwen, Groq-hosted chat, OpenAI-hosted chat, and deterministic fallback

Runtime Flow
------------
Student query
-> Safety agent
-> Planner/router agent
-> Student adaptation agent
-> Retrieval/tool/quiz/grading branch
-> Tutor/Quiz/Grader response agent
-> Checker agent
-> Trace writer
-> Student answer plus retrieved sources and backend trace

Implemented Components
----------------------
- Prompt design: src/llm/prompts.py
- LLM provider routing: src/llm/client.py and src/llm/model_registry.py
- RAG: src/rag/ with BM25 offline retrieval, optional semantic retrieval, Tavily/ddgs online retrieval, and hybrid merging
- Fine-tuning/PEFT: src/finetuning/, scripts/04_train_lora.py, notebooks/03_finetuning_complete.ipynb, outputs/finetune/qwen_0_5b_lora_adapter_salma/
- Imported Fatma adapters: outputs/finetune/final_model_fatma/, mistral_lora_adapter_fatma/, phi3_lora_model_fatma/, qlora_model_fatma/, qwen_lora_model_fatma/
- Imported Khadija adapter: outputs/finetune/qwen_0_5b_lora_adapter_khadija/
- Tools/function calling: src/tools/
- Multi-agent setup: src/agents/graph.py plus specialized agents in src/agents/
- Evaluation: src/evaluation/ and outputs/evaluation/
- Ethics/safety: src/agents/safety_agent.py and docs/ethics_safety.md
- GUI: app.py

Model Provider Behavior
-----------------------
- Fine-tuned Qwen LoRA is preferred when local transformer/PEFT dependencies and adapter files are available.
- Base Qwen is available when local transformer dependencies and model files are available.
- Groq is available when GROQ_API_KEY is configured; default model is llama-3.1-8b-instant.
- OpenAI is available when OPENAI_API_KEY is configured; default model is gpt-4o-mini.
- Deterministic fallback is always available for no-key demos and tests.

Deployment Notes
----------------
- Streamlit Community Cloud should use requirements.txt.
- ChromaDB and fine-tuning dependencies are intentionally optional to avoid cloud protobuf/import failures.
- Local semantic retrieval can be enabled with requirements_semantic.txt.
- Local LoRA training/inference dependencies can be installed from requirements_finetune.txt.

Validation
----------
Use python3 -m pytest tests/ -q for the current verification suite.
